# 四格漫画模板 (4コマ / Yonkoma)

四格漫画是日本传统漫画形式，以四个格子讲述一个完整的小故事，结构固定为「起承転結」。

---

## 基本结构

### 起承転結 (Ki-Sho-Ten-Ketsu)

```
┌─────────────┐
│     起      │  ← 第1格: 起 (Ki) - 设定/开始
│   Setup     │     介绍场景、角色、情境
├─────────────┤
│     承      │  ← 第2格: 承 (Sho) - 发展
│ Development │     延续第一格，情节推进
├─────────────┤
│     転      │  ← 第3格: 転 (Ten) - 转折
│   Twist     │     意外变化，笑点铺垫
├─────────────┤
│     結      │  ← 第4格: 結 (Ketsu) - 结局
│ Conclusion  │     落点/笑点/余韵
└─────────────┘
```

### 阅读方向

**标准四格**: 从上到下，右到左（如果一页两列）

```
单列四格:       双列四格（一页）:
┌───┐           ┌───┐ ┌───┐
│ 1 │           │ 1 │ │ 5 │
├───┤           ├───┤ ├───┤
│ 2 │           │ 2 │ │ 6 │
├───┤           ├───┤ ├───┤
│ 3 │           │ 3 │ │ 7 │
├───┤           ├───┤ ├───┤
│ 4 │           │ 4 │ │ 8 │
└───┘           └───┘ └───┘
 ↓               右列   左列
```

## 四格漫画类型

### 日常搞笑型

**结构**:
- 起: 日常场景
- 承: 平常发展
- 転: 意外/吐槽点
- 結: 笑点落地

**示例主题**:
- 学校日常
- 家庭趣事
- 宠物互动
- 工作吐槽

### 恋爱甜蜜型

**结构**:
- 起: 两人相处
- 承: 互动发展
- 転: 心动瞬间
- 結: 甜蜜或糖炸

**示例主题**:
- 情侣日常
- 暗恋心事
- 告白场景
- 吃醋互动

### 治愈温馨型

**结构**:
- 起: 平静日常
- 承: 温馨互动
- 転: 小感动
- 結: 温暖余韵

**示例主题**:
- 亲子互动
- 友情片段
- 小确幸
- 季节感悟

## 笑点技法

### 反转型

```
起承: 建立预期
転: 反转预期
結: 强化反转效果

例:
起: 主角: 「今天一定要认真学习！」
承: 打开书本，表情认真
転: 3小时后...（书没翻页）
結: 「我思考人生思考了3小时」
```

### 吐槽型

```
起承: 角色A做奇怪的事
転: 角色A继续/升级
結: 角色B吐槽

例:
起: A在认真写什么
承: A写得更投入了
転: A: 「完成了！我的自传！」（才5页）
結: B: 「你人生也太薄了」
```

### 天然呆型

```
起承: 正常对话
転: 角色的天然回应
結: 众人反应/沉默

例:
起: 「快要期末考了...」
承: 「你复习了吗？」
転: A: 「复习？考试不是用来检验真实水平的吗？」
結: [全员沉默] [A一脸无辜]
```

### 重复升级型

```
起: 事件发生
承: 同样的事再次发生
転: 第三次...更夸张
結: 角色崩溃/接受现实

例:
起: 手机掉地上（没坏）
承: 又掉了（没坏）「还好」
転: 轻轻放在桌上...滑落摔碎
結: [无声呐喊]
```

## 整页生成指南

本节用于将企划阶段设计的四格内容合并为一张完整的漫画页图片生成。

### 整页生成原理

整页生成将四个格子的内容合并为一个结构化提示词：
- 使用 `panel layout` 描述格子布局
- 按阅读顺序（从上到下）描述每格内容
- 统一风格关键词放在最后

### 整页提示词基础结构

```
4-koma manga page, 4 panel vertical layout,
Japanese comedy manga style,

panel 1 (top): [镜头] [角色] [动作] [表情] [场景],
panel 2: [镜头] [角色] [动作] [表情],
panel 3: [镜头] [角色] [动作] [表情] [转折效果],
panel 4 (bottom): [镜头] [角色] [动作] [表情] [落点效果],

cute character design, expressive reactions,
black and white, screentone shading, clean panel borders
--ar 9:16 --v 7
```

### 推荐宽高比

| 布局 | 宽高比 | 说明 |
|------|--------|------|
| 单列四格 (标准) | `--ar 9:16` | 竖版，最常用 |
| 单列四格 (宽松) | `--ar 2:5` | 更宽松的竖版 |
| 横版四格 | `--ar 16:9` | 一行排列 |

### 起承転結整页模板

```

cute expressive characters, comedic timing,
```

### 角色一致性技巧

在整页生成中保持角色一致性：

```

panel 1: young girl with long black hair and ribbon, [场景],
panel 2: same young girl with long black hair and ribbon, [动作],
panel 3: same young girl with long black hair and ribbon, [表情],
panel 4: same young girl with long black hair and ribbon, [反应],

consistent character design throughout all panels,
black and white manga style
--ar 9:16 --v 7
```

**关键词重复法**: 在每个panel描述中重复核心角色特征（发型、配饰、服装特点）

### 整页生成 vs 逐格生成对比

| 方面 | 整页生成 | 逐格生成 |
|------|----------|----------|
| 一致性 | 较高（同一生成） | 需要额外处理 |
| 控制度 | 整体控制 | 每格精细控制 |
| 适用场景 | 快速草稿、风格统一 | 重要页面、精细调整 |
| 后期编辑 | 需整体重生成 | 可单格替换 |

**建议工作流**:
1. 企划阶段：逐格详细设计
2. 草稿阶段：使用整页生成快速预览
3. 定稿阶段：根据需要选择整页或逐格

## 连载四格技巧

### 角色固定

四格漫画适合固定角色组合:

| 组合 | 说明 | 例子 |
|------|------|------|
| 吐槽组 | 天然+吐槽役 | A做蠢事，B吐槽 |
| 日常组 | 多人日常 | 学生/同事/室友 |
| 反差组 | 外表vs内心 | 冷酷实则可爱 |
| CP组 | 恋人/暧昧 | 甜蜜日常 |

### 系列命名

```
《[角色名]的日常》
《[地点]物语》
《[主题]笔记》
《那个[角色]》
```

## 检查清单

```markdown
□ 起承転結结构是否完整？
□ 転（转折）是否足够意外？
□ 結（结局）落点是否有力？
□ 表情是否足够夸张清晰？
□ 对话是否精炼（每格1-2句）？
□ 四格之间是否有连贯性？
□ 单独看每格是否能理解？
```
