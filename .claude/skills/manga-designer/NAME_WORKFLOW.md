# ネーム制作流程 (Name/Storyboard Workflow)

ネーム（分镜稿）是日本漫画的设计蓝图，是漫画家与编辑沟通的核心文档。本文档详细说明ネーム的制作流程和技巧。

---

## 什么是ネーム？

### 定义

**ネーム (Name)** 是日本漫画业界对分镜稿的称呼，源自英语"name"（命名/定义）。它是漫画的设计蓝图，决定了：

- 画格（コマ）的分配与大小
- 角色在每格中的展现方式
- 台词的配置与阅读顺序
- 整体节奏与翻页体验

### 工作流程位置

```
企划 → キャラ設定 → ネーム → 下絵 → ペン入れ → 仕上げ
(策划)   (角色设定)   (分镜)   (草稿)  (描线)    (完稿)
              ↑
        【本流程重点】
```

### ネーム的特点

| 特点 | 说明 |
|------|------|
| 简略画面 | 不需要精细绘画，只需表达意图 |
| 完整台词 | 对话必须完整写出 |
| 格子布局 | 明确的格子分割 |
| 阅读流 | 清晰的RTL阅读顺序 |
| 可迭代 | 便于与编辑讨论修改 |

## ネーム制作步骤

### Step 1: 台词先行（セリフ先行）

**原则**: 日漫读者先看台词，再看画面。因此分镜设计从台词开始。

#### 1.1 提取关键台词

从剧本中提取每个场景的关键对话：

```markdown
## 第1场景 - 教室

林浩（独白）：「又是平凡的一天...」
班长：「林浩！你又迟到了！」
林浩：「抱歉抱歉！」
（新转学生出现）
老师：「给大家介绍一位转学生...」
雨晴：「大家好，我是苏雨晴。」
林浩（内心）：「这个女孩...好像在哪见过？」
```

#### 1.2 台词分配到格子

```markdown
## 第1页 台词分配

格1: [无对话] - 学校外景建立镜头
格2: 「又是平凡的一天...」 - 独白框
格3: 「林浩！你又迟到了！」 - 班长
格4: 「抱歉抱歉！」 - 主角回应
格5: [悬念格] 「给大家介绍一位转学生...」 - 老师
```

#### 1.3 台词阅读顺序检查

确保台词按RTL顺序自然阅读：

```
格内阅读顺序：右上 → 左上 → 右下 → 左下

         ┌────────────────┐
         │ ①「台词A」    │
         │        ②「台词B」│
         │ ③「台词C」    │
         └────────────────┘
```

---

### Step 2: 格子分配（コマ割り）

#### 2.1 确定每页格子数

| 场景类型 | 建议格数/页 | 说明 |
|----------|-------------|------|
| 建立镜头 | 2-3 | 大格展示环境 |
| 对话场景 | 4-6 | 标准密度 |
| 动作场景 | 5-8 | 小格快节奏 |
| 高潮场景 | 1-3 | 大格强调 |
| 情感高潮 | 2-4 | 留白重要 |

#### 2.2 格子大小分配

**节奏控制公式**:

```
重要度 ∝ 格子大小
节奏 ∝ 1/格子大小

大格 = 重要 + 慢节奏
小格 = 次要 + 快节奏
```

**典型分配**:

```
一页标准分配（6格）:
┌─────────────────────────┐
│ ┌────────┐ ┌─────┐     │
│ │  大格  │ │中格 │ ←主场景│
│ │ (40%) │ │(25%)│     │
│ └────────┘ └─────┘     │
│ ┌────┐ ┌────┐ ┌────┐  │
│ │小格│ │小格│ │中格│ ←推进│
│ │10%│ │10%│ │15%│     │
│ └────┘ └────┘ └────────┘│
└─────────────────────────┘
```

#### 2.3 翻页钩子（めくり）

**规则**: 每页右下角格子必须有"吸引翻页"的元素

| 钩子类型 | 示例 | 效果 |
|----------|------|------|
| 问题悬念 | 「那是...什么!?」 | 引发好奇 |
| 动作起势 | 准备出拳的瞬间 | 期待下一刻 |
| 新角色 | 神秘人影出现 | 想知道是谁 |
| 情感反转 | 惊讶表情特写 | 想知道原因 |
| 威胁暗示 | 危险靠近 | 担心角色 |

### Step 3: 阅读流设计

#### 3.1 RTL基本流向

```
单页阅读流:
          START
            ↓
    ┌───────┴───────┐
    │ ①  →→→   ②  │
    │ ↓            ↓│
    │ ③  ←←←←←←←←┘│
    │ ↓              │
    │ ④  →→→   ⑤  │
    │             ↓ │
    └───────────→END│
```

#### 3.2 视线引导技巧

| 技巧 | 方法 | 效果 |
|------|------|------|
| 角色朝向 | 角色面向下一格方向 | 自然引导 |
| 台词位置 | 台词指向阅读方向 | 视线跟随 |
| 动作线 | 动作指向下一格 | 动态引导 |
| 留白 | 在阅读方向留空 | 无障碍流动 |

#### 3.3 跨格连贯

```markdown
## 动作连续示例

格1: 准备出拳 (右)
      ↓ [动作连接]
格2: 出拳中途 (中)
      ↓ [动作连接]
格3: 命中瞬间 (左)
      ↓ [反应连接]
格4: 对手后退 (大格)

视觉连续性: 拳头轨迹贯穿4格
```

### Step 4: 特殊技法应用

#### 4.1 破格（はみ出し）

角色或物体突破格子边界：

```
使用场景:
├── 动作强调: 拳头/武器突出
├── 角色登场: 全身突破展示
├── 情绪爆发: 表情突破边界
└── 动感表现: 运动轨迹溢出

ネーム标注:
[格3 - 破格] 主角拳头突破右边界
          → 强调攻击力度
```

#### 4.2 见开き（跨页）

两页合并为一个大画面：

```
使用规则:
├── 整个短篇最多1-2次
├── 保留给最高潮场景
├── 通常用于第7-10页（转折高潮）
└── 需要考虑装订线

ネーム标注:
[第7-8页 见开き]
最终决战画面，主角vs反派对峙
左页: 主角冲锋
右页: 反派迎战
中央: 能量碰撞
```

#### 4.3 倾斜格子

格子边框倾斜，表现紧张/不安：

```
使用场景:
├── 战斗激烈时刻
├── 心理动摇
├── 时空扭曲
└── 速度感强调

角度建议: 5-15度倾斜
过度倾斜会影响阅读
```

---

### Step 5: ネーム文档格式

#### 5.1 标准ネーム文档

```markdown
# 《作品标题》ネーム

## 基本信息
- 页数: 16页
- 格子总数: 约70格
- 类型: 少年漫画

---

## 第1页 (RTL阅读顺序)

### 页面概述
- 功能: 起·开篇，建立世界
- 格数: 4格
- 节奏: 中等

### 格1 (右上 - 大格 40%)
- **类型**: 场景建立
- **镜头**: 远景俯瞰
- **内容**: 城市全景，清晨
- **台词**: [无]
- **效果**: 无
- **备注**: 建立故事舞台

### 格2 (右中 - 中格 25%)
- **类型**: 角色介绍
- **镜头**: 中景
- **内容**: 主角在公寓窗边
- **台词**:
  - 独白「又是新的一天开始了...」
- **角色**: [hero_001]
- **效果**: 网点（窗户光线）
- **表情**: 平静带一丝无聊

### 格3 (左上 - 小格 15%)
- **类型**: 细节过渡
- **镜头**: 特写
- **内容**: 手机闹钟显示 7:30
- **台词**: [无]
- **效果**: 无
- **备注**: 日常生活细节

### 格4 (左下 - 中格 20%) [めくり]
- **类型**: 悬念钩子
- **镜头**: 特写
- **内容**: 手机来电，显示「未知号码」
- **台词**: [无]
- **效果**: 轻微集中线
- **翻页引导**: ✓ 悬念 - 谁在打电话？

---

## 第2页

### 页面概述
- 功能: 起·事件触发
- 格数: 5格
- 节奏: 中→快

[继续...]
```

#### 5.2 简化ネーム速记

用于快速草稿阶段：

```
P1: 城市远景 | 主角窗边[独白] | 手机7:30 | 来电[悬念→]
P2: 接电话 | 神秘声音[台词] | 惊讶脸 | 摔倒[搞笑] | 收拾出门[快节奏→]
P3: 学校门口 | 遇到班长[对话×3] | 转学生登场[大格→]
...
```

## 编辑审核要点

### 节奏检查

```markdown
□ 起: 是否在2页内建立世界和角色？
□ 承: 每2-3页是否有小高潮？
□ 転: 高潮是否有足够版面？（见开き？）
□ 結: 结尾是否有余韵，不仓促？
```

### 技术检查

```markdown
□ RTL阅读流是否顺畅？
□ 每页是否有めくり（翻页钩子）？
□ 格子大小变化是否支持节奏？
□ 破格使用是否适度（不超过3次/页）？
```

### 台词检查

```markdown
□ 台词是否精炼（每格不超过2-3句）？
□ 台词位置是否支持阅读流？
□ 关键台词是否有足够视觉强调？
```

## 常见问题与解决

### 问题1: 页面太密集

**症状**: 格子太多，画面拥挤
**解决**:
- 删除非必要格子
- 合并相似内容格子
- 考虑扩展页数

### 问题2: 节奏拖沓

**症状**: 读起来很慢，没有紧张感
**解决**:
- 减少对话格，增加动作格
- 使用更多小格快推
- 删减解释性内容

### 问题3: 高潮不够高

**症状**: 转折点没有冲击力
**解决**:
- 给高潮场景更大版面
- 考虑使用见开き
- 增加前期铺垫压抑

### 问题4: 阅读混乱

**症状**: 读者不知道看哪里
**解决**:
- 检查格子排列是否符合RTL
- 使用角色朝向引导视线
- 简化复杂布局

## 与 AI 生成集成

### ネーム → 提示词转换

每个格子描述可直接转换为 prompt-generator 输入：

```markdown
## 格子描述
- 镜头: 中景，仰视
- 角色: [hero_001]
- 动作: 出拳
- 效果: 集中线、破格

## 转换为提示词输入
```
调用 /prompt-generator
- 任务: text-to-prompt
- 模式: Standard
- 输入: "漫画面板，中景仰视角度，[hero_001角色特征]，
        出拳动作，集中线效果，拳头突破边框"
- 风格: shounen manga, black and white
```

### 批量生成工作流

```
1. 完成完整ネーム文档
2. 标记需要生成的关键帧（通常是大格、高潮格）
3. 为每个标记格调用 prompt-generator
4. 可选: 调用 generating-images 生成预览
5. 根据生成结果调整ネーム
```
