# Skill Orchestrator — 技能编排器

## Overview

**当此技能被调用时，你必须立即执行以下工作流，不要只描述它。**

```
用户复杂目标
    ↓
[分析阶段] 理解意图、分解任务
    ↓
[规划阶段] 确定技能列表、执行顺序、依赖关系
    ↓
[确认阶段] 向用户展示执行计划，等待确认 ★
    ↓
[执行阶段] Spawn agent-teammates with skills
    ↓
[聚合阶段] 等待 agent-teammates 分别返回结果并整合各技能输出
    ↓
最终交付物
```

## 分析用户目标

对收到的用户输入进行分析（不要只输出分析步骤，要实际执行）：

1. **意图提取** — 明确用户最终目标是什么
2. **约束条件** — 时间、格式、风格等限制
3. **期望交付物** — 用户想要什么具体结果

输出格式：

```text
## 分析结果
- **最终目标**：[具体描述]
- **约束条件**：[列出约束]
- **期望交付物**：[描述交付物]
```

## 任务分解

将复杂目标分解为原子步骤：

1. 列出所有需要的原子操作（3-8步）
2. 识别步骤间的依赖关系
3. 确定哪些可以并行执行

输出格式：

```text
## 任务分解
1. [步骤描述] → 依赖：[无/前置步骤]
2. [步骤描述] → 依赖：[无/前置步骤]
...
```

## 技能发现

根据任务分解结果，选择需要的技能：查看 [references/known_skills.md](references/known_skills.md) 以获取技能发现的相关信息

## 执行前确认

**使用 AskUserQuestion 工具向用户展示执行计划并等待确认。**

向用户展示：

- 将要调用的技能列表
- 各技能的职责
- 执行顺序（并行/串行）
- 各 teammates 的任务描述

示例问题：
"我将按以下计划执行：1) 使用 XX 技能做... 2) 使用 YY 技能做... 是否继续？"

## 执行与聚合

获得用户确认后：

1. **Spawn teammatess**：使用 Agent 工具 spawn 多个 teammates，每个携带对应技能
2. **收集结果**：等待所有 teammates 完成
3. **整合输出**：将各 teammates 的结果合并为统一的最终交付物

输出格式：

```text
## 执行结果
[各 teammates 结果整合]
---
## 最终交付物
[统一格式的最终输出]
```
