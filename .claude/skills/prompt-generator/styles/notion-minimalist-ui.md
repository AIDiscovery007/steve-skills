# Notion-Style Minimalist UI Design

## Style Overview

A modern, clean, and functional UI design style characterized by extreme minimalism, generous whitespace, and clarity-first information architecture. Inspired by productivity tools like Notion, Linear, and contemporary SaaS applications. Perfect for dashboard mockups, app interface designs, productivity tool visuals, and professional software presentations.

**Visual Characteristics**:

- Pure white or light neutral backgrounds
- Simple linear icons with consistent stroke width
- Clear typographic hierarchy with sans-serif fonts
- Subtle borders and gentle shadows
- Generous spacing and breathing room
- Rounded corners for softer feel
- Monochromatic color scheme with minimal accent colors
- Card-based layouts and organized sections
- Hand-drawn annotation arrows for explanatory notes (optional enhancement)

**Use Cases**:

- SaaS application interface design mockups
- Productivity tool and dashboard presentations
- Project management software UI concepts
- Knowledge management system visuals
- Professional portfolio case studies
- Tech product landing page screenshots
- App Store preview images
- Design system documentation

## Standard Parameter Configuration

### Basic Configuration

```text
--v 6 --ar [ratio] --stylize 0-50 --style raw
```

**Parameter Explanation**:

| Parameter | Recommended Value | Rationale |
|-----------|------------------|-----------|
| Version | `--v 6` | Best for clean UI rendering and text-like elements |
| Aspect Ratio | `--ar 16:9` / `--ar 4:3` | Widescreen for desktop interfaces, 4:3 for balanced view |
| Stylization | `--stylize 0-50` | Very low stylization maintains clean, functional aesthetic |
| Style Mode | `--style raw` | Prevents artistic interpretation, keeps design minimal |

### Parameter Adjustments for Different Scenarios

**Desktop Dashboard**:

```text
--ar 16:9 --stylize 20
```

- 16:9 suits desktop/laptop screens
- Minimal stylization for functional clarity

**Mobile App Interface**:

```text
--ar 9:16 --stylize 30
```

- Vertical orientation for mobile screens
- Slightly higher stylization for polish

**Square Social Media Preview**:

```text
--ar 1:1 --stylize 40
```

- Square format for Instagram/Twitter
- Moderate stylization for visual appeal

**Presentation Slide**:

```text
--ar 16:9 --stylize 10 --q 2
```

- Standard presentation ratio
- Minimal stylization, high quality for clarity

## Common Adjustment Needs

### If Interface Looks Too Cluttered

**Simplify**:
- Reduce component descriptions
- Add: `extra generous whitespace, minimal UI elements, focus on essential content`
- Lower --stylize to 0-10
- Specify: `maximum simplicity, breathing room prioritized`

---

### If Text Hierarchy Is Unclear

**Enhance hierarchy**:
- Add: `strong typographic hierarchy, bold large headings, clear size contrast between levels`
- Specify exact size differences: `32px headings, 16px body, 12px labels`
- Add: `visual weight through font weight variation`

---

### If Design Looks Too Plain/Boring

**Add subtle interest** (while maintaining minimalism):
- Add: `subtle accent color highlights, soft pastel category colors`
- Include: `micro-interactions, gentle hover states, smooth transitions`
- Use: `subtle background tints (#F9F9F9) for sections`
- Slightly increase --stylize to 40-50

---

### If Colors Are Needed for Categories

**Implement color system**:
- Add: `color-coded categories with pastel backgrounds (soft pink, light blue, mint green, pale yellow)`
- Specify: `minimal saturation, soft pastel tones, each category with distinct color`
- Ensure: `colors are secondary to content, not dominant`

---

### If Icons Look Inconsistent

**Standardize icons**:
- Add: `unified icon system, consistent 2px stroke weight across all icons, same style family`
- Specify: `16x16px or 24x24px icon grid, geometric consistency`
- Note: `outline style for all icons, no mixing filled and outline`

---

### If Shadows Are Too Heavy/Light

**Adjust elevation**:

**Too heavy** → Lighten:
```text
minimal shadows (0px 1px 2px rgba(0,0,0,0.03)), barely perceptible depth
```

**Too light** → Strengthen:
```text
noticeable but subtle shadows (0px 4px 12px rgba(0,0,0,0.08)), clear elevation
```

**Remove completely**:
```text
no shadows, border-based separation only, completely flat design
```

## Important Notes

### ✅ Should Do

1. **Keep stylization very low**: 0-50 range to maintain functional clarity
2. **Use --style raw**: Prevents artistic interpretation
3. **Specify exact colors**: Use hex codes or specific color names for consistency
4. **Define clear hierarchy**: Mention font sizes, weights, and text color variations
5. **Emphasize whitespace**: Explicitly call out spacing and padding
6. **Consistent icon style**: Specify stroke width and style (outline vs filled)
7. **List specific components**: Name the UI elements you need (buttons, cards, etc.)

### ❌ Should Avoid

1. **Don't use --niji**: Will add anime/illustration style, unsuitable for UI mockups
2. **Don't stylize too high**: Above 100 will add unwanted artistic elements
3. **Don't describe "beautiful" or "stunning"**: Focus on functional terms like "clean," "organized," "clear"
4. **Don't mix styles**: Avoid combining with illustrative or artistic descriptors
5. **Don't add unnecessary decoration**: Every element should serve a functional purpose
6. **Don't overcomplicate**: Minimal UI means less is more
7. **Don't forget to exclude**: Use negative prompts to avoid gradients, textures, etc.
